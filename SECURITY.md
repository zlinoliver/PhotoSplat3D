# Security Policy

## Reporting a Vulnerability

Please report security issues privately before public disclosure.

Include:

1. Affected version/commit
2. Impact summary
3. Reproduction steps or proof-of-concept
4. Suggested mitigation (if any)

If contact details are not yet published, open a private channel first (for example, via repository owner contact) and avoid filing public issues for vulnerabilities.

## Scope

This policy covers the source code in this repository and release artifacts produced from it.
